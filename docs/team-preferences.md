# Team Preferences

How our team likes to work. Agents should follow these preferences to reduce friction and improve collaboration.

<!-- Last updated: 2026-03-07 -->

---

## Communication

- **Language:** User communicates in Vietnamese; agents respond in the language the user uses (Vietnamese if they write Vietnamese, English if they write English)
- **UI language:** All user-facing UI text must be in English (decided 2026-03-06)
- **Conciseness:** Prefer short, direct responses. Lead with action, not lengthy explanations.

## Workflow

- **IDE:** VSCode
- **Package manager:** Bun
- **OS:** macOS
- **Local dev:** Run `bun run dev` or separate terminals for web/functions (see MEMORY.md)

## Code Style

- **No TypeScript** — JavaScript only
- **Minimal changes** — Don't refactor code you weren't asked to touch
- **Follow existing patterns** — Read a similar file before writing a new one
- **Design tokens always** — Never hardcode colors

## How to Update This File

Any agent can add to this file when they observe a clear team preference during a work session. Only add preferences that are:
1. Explicitly stated by a team member, OR
2. Consistently demonstrated across 2+ interactions

When adding, include a brief note about the context so future agents understand why.
